# Reference: Pull Requests

This project does not require or enforce any specific pull request labeling scheme.
The table below documents common conventions which may be used as appropriate.

|   Label  |                                Typical Meaning                                |
|----------|-------------------------------------------------------------------------------|
| infra    | Changes to infrastructure (build tools, CI/CD, GitHub Actions, configs, etc.) |
| deps     | Dependency updates (e.g., changing requirements.txt or updating packages)     |
| chore    | Routine or non-functional changes (e.g., renaming, cleanup, formatting)       |
| docs     | Documentation-only changes (README.md, REF_DEV.md, etc.)                      |
| test     | Adding or updating tests                                                      |
| feat     | A new feature                                                                 |
| fix      | A bug fix                                                                     |
| refactor | Code restructuring without functional change                                  |
| release  | Tagged releases or version bumps                                              |
| security | Security-related patches or dependencies                                      |
